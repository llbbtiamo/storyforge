# StoryForge 架构设计文档

> 面向结构化小说创作的 AI 写作平台

## 1. 项目概述

### 1.1 产品定位

StoryForge 是一个面向**个人作者**的 AI 小说写作平台。通过世界观设定、角色档案、剧情大纲、风格约束等前置配置，驱动 AI 自动生成连贯、可控的长篇小说内容。

### 1.2 核心价值

- **结构化创作**：不是简单的提示词续写，而是基于完整世界观体系的自动化创作
- **AI 生成 + 人工审核**：章节级自动生成，作者审核修改后确认
- **世界观复用**：用户自建的世界观可跨项目复用

### 1.3 商业模式

**订阅 + 额度混合制**：

| 层级 | 月费 | 包含额度 | 功能权限 |
|------|------|---------|---------|
| 免费用户 | ¥0 | 少量体验额度 | 基础功能，限制项目数 |
| 标准 VIP | ¥29/月 | 中等额度 | 预设世界观模板、更多项目 |
| 高级 VIP | ¥79/月 | 大量额度 | 自定义模型接入、API 权限 |
| 额度包 | 按量购买 | 灵活补充 | 所有层级均可购买 |

### 1.4 开源策略

- **社区版（开源）**：自托管，用户自带模型 API Key，无额度限制，无预设模板
- **商业 SaaS 版**：托管服务，订阅制，预设模板，管理后台

### 2.2 架构策略：单体优先 → 渐进拆分

**MVP 阶段**：所有服务模块打包在一个 Spring Boot 应用中，通过 Maven 多模块保持代码边界清晰。

**扩展阶段**：当用户量增长后，将 AI 适配层、计费服务等按需拆分为独立微服务，引入 Spring Cloud Alibaba 组件：

- Nacos：服务注册与配置中心
- Spring Cloud Gateway：API 网关
- Sentinel：流控与熔断
- Seata：分布式事务（如需）

## 4. 数据模型设计

### 4.1 ER 关系总览

```
用户(User) 1──N 项目(Project) 1──N 章节(Chapter)
                    │
                    ├── N 世界观设定(WorldSetting)
                    ├── N 角色档案(CharacterProfile)
                    ├── N 剧情大纲(PlotOutline)
                    ├── 1 风格约束(StyleConstraint)
                    └── N 知识库条目(KnowledgeEntry)

用户(User) 1──N 世界观模板(WorldTemplate)  ← 可跨项目复用
用户(User) 1──1 订阅信息(Subscription)
用户(User) 1──1 额度账户(QuotaAccount)

角色(Character) N──N 角色关系(CharacterRelation)
```

## 5. 核心 API 设计

### 5.1 API 规范

- RESTful 风格，统一前缀 `/api/v1`
- 统一响应格式：`{ code, message, data }`
- JWT 认证方式，Token 通过 Header `Authorization: Bearer <token>` 传递
- SpringDoc 自动生成 OpenAPI 文档

### 5.2 核心 API 清单

#### 认证模块

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/auth/register` | 用户注册 |
| POST | `/api/v1/auth/login` | 用户登录 |
| POST | `/api/v1/auth/refresh` | 刷新 Token |
| GET  | `/api/v1/auth/me` | 获取当前用户信息 |

#### 项目模块

| 方法 | 路径 | 说明 |
|------|------|------|

| GET  | `/api/v1/projects/{id}` | 获取项目详情 |
| PUT  | `/api/v1/projects/{id}` | 更新项目 |
| DELETE | `/api/v1/projects/{id}` | 删除项目 |

#### 世界观模块

| 方法 | 路径 | 说明 |

## 6. AI 适配层设计

### 6.1 核心接口

```java
/**
 * 模型适配器统一接口（SPI）
 */
public interface ModelAdapter {

    /** 提供商标识 */
    String getProvider();

    /** 支持的模型列表 */
    List<String> getSupportedModels();

    /** 文本生成（流式） */
    Flux<String> generateStream(GenerationRequest request);

    /** 文本生成（阻塞） */
    String generate(GenerationRequest request);

    /** 估算 Token 消耗 */
    int estimateTokens(String text);
}
```

### 6.2 Prompt 编排引擎

章节生成不是简单地调用一次 LLM，而是一个**多步骤编排**过程：

```
1. 收集上下文
   ├── 加载世界观设定（精简摘要）
   ├── 加载相关角色档案
   ├── 加载前 N 章内容摘要（记忆窗口）
   ├── 加载当前大纲节点要求
   └── 加载风格约束规则

2. 构建 Prompt
   ├── System Prompt: 世界观 + 角色 + 风格规则
   ├── Context: 前文摘要 + 关键剧情线索
   └── Instruction: 本章生成要求

3. 调用模型生成

4. 后处理
   ├── 内容质量检查（字数、连贯性）
   ├── 一致性校验（角色名、设定矛盾检测）
   └── 存储结果 + 扣减额度
```

### 6.3 上下文管理策略

长篇小说的核心挑战是**上下文窗口有限**。解决方案：

- **滑动摘要窗口**：每章生成完成后自动生成该章摘要，供后续章节参考
- **关键事件索引**：维护剧情关键事件的时间线索引
- **角色状态追踪**：随剧情推进更新角色当前状态（位置、情绪、关系变化）
